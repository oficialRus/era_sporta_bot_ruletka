# Исправление: «Подключение не защищено» и 502 Bad Gateway

## Что сделано в конфиге

В `/etc/nginx/sites-available/bot-wheel-era-sporta.conf` добавлен блок **HTTPS** (порт 443) с вашим сертификатом Let's Encrypt и проксированием на:
- **/** → `http://127.0.0.1:5173` (веб-приложение)
- **/api/** → `http://127.0.0.1:8080` (API)

Запросы по HTTP (порт 80) перенаправляются на HTTPS.

Раньше в конфиге был только порт 80, поэтому при открытии **https://** bot-wheel... не находился правильный виртуальный хост и возникали 502 и предупреждение о подключении.

## Что сделать на сервере (одной командой)

```bash
sudo nginx -t && sudo systemctl reload nginx
```

- `nginx -t` — проверит конфиг.
- `systemctl reload nginx` — применит изменения.

## Проверка

1. В браузере откройте:  
   **https://bot-wheel.era-sporta-apsheronsk.ru**  
   Должна открыться страница рулетки без предупреждения и без 502.

2. Проверка API:  
   **https://bot-wheel.era-sporta-apsheronsk.ru/api/prizes**  
   Должен вернуться JSON с призами.

3. В Telegram откройте бота и нажмите «Открыть приложение» — должно открываться приложение по HTTPS без ошибок.

## Если 502 остаётся

Убедитесь, что на сервере запущены оба сервиса:

```bash
cd /root/era_sporta_bot_ruletka
./status.sh
```

Должны быть запущены **api** и **web**. Если нет:

```bash
./stop.sh
./start.sh
```

После этого снова выполните:

```bash
sudo systemctl reload nginx
```
